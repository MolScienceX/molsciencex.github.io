# MolScience 项目白皮书

## 面向人工智能时代的化学分子数据库构建与智能化应用

------

## 一、项目概述

**MolScience** 是由中国人民大学化学与生命资源学院本科生团队发起的科研项目，致力于构建一个**标准化、可扩展、智能化的化学分子数据库基础设施**。

项目以人工智能时代科学研究范式转变为背景，围绕“数据—模型—应用”三位一体展开，目标是推动分子数据库从传统“数据存储工具”向“智能知识引擎”升级。

核心目标包括：

- 构建**自主可控的分子数据库体系**
- 实现**分子数据的标准化与高效检索**
- 打造**数据驱动的分子性质预测与设计能力**
- 建立**自然语言驱动的智能化交互范式**

该项目已完成初步平台开发，拥有**1.2亿分子数据基础与4万+行代码实现** ，并正在持续推进系统化建设。

------

## 二、研究背景与问题提出

### 1. 数据驱动科研范式的兴起

随着人工智能的发展，科学研究正进入“第四范式”——数据密集型科学发现。数据不再仅用于验证理论，而成为知识生成的核心来源 。

在化学领域：

- AlphaFold 等成果表明 AI 已可直接推动科学突破
- 分子设计、催化路径探索等逐渐转向数据驱动

结论：

> **高质量分子数据 + 高效数据库 = AI化学的基础设施**

------

### 2. 数据“卡脖子”风险

当前全球核心分子数据库（如 SciFinder、PubChem 等）主要由欧美机构控制，国内科研高度依赖外部数据源 。

最新趋势：

- NIH 限制中国访问关键数据库
- PubChem 出现访问限制趋势
- 生物医药与化学数据已成为“战略资源”

影响：

- AI模型训练数据受限
- 科研连续性存在风险
- 数据主权问题凸显

------

### 3. 现有数据库的核心缺陷

当前分子数据库普遍存在三类问题：

#### （1）架构问题

- 主要基于关系型数据库
- 不支持图结构与向量检索
- 难以适配AI应用

#### （2）数据问题

- 标准不统一
- 数据质量参差不齐
- 缺乏系统治理机制

#### （3）智能化缺失

- 依赖精确匹配（如SMILES）
- 无法支持自然语言查询
- 与大模型脱节

结论：

> **现有数据库本质上是“人用工具”，而非“AI原生系统”**

------

## 三、项目总体目标

MolScience 旨在构建一个**AI原生的分子数据库系统**，具体目标如下：

### 数据层

- 构建**1.2亿级分子数据库**
- 实现统一标准化表示（SMILES / InChI）

### 平台层

- 开发Web平台并对外开放
- 建立跨高校数据共享社区

### 算法层

- 实现分子性质预测
- 构建相似性搜索系统
- 支持分子设计与合成路径规划

### 交互层

- 构建基于大模型的自然语言问答系统

------

## 四、核心技术体系

### 1. 多模态数据库协同架构（核心创新）

项目提出三类数据库协同架构：

| 类型         | 功能                               |
| ------------ | ---------------------------------- |
| 关系型数据库 | 存储结构化属性（分子式、分子量等） |
| 图数据库     | 表示反应网络与合成路径             |
| 向量数据库   | 支持分子相似性检索                 |

优势：

- 同时支持**精确查询 + 关系分析 + 语义检索**
- 面向AI任务原生设计

该架构突破传统单一数据库模式，是项目的关键创新点之一 。

------

### 2. 数据处理与标准化体系

数据处理流程包括：

- 多源数据整合（PubChem、文献等）
- 标准化表示（SMILES / InChI）
- 数据清洗：
  - 去重
  - 缺失值填补
  - 异常值检测
  - 去除盐离子/溶剂等非关键结构
- 数据一致性校验

目标：

> 构建**高质量、可建模的分子数据基础**

------

### 3. 分子建模与预测系统

技术路径：

**数据 → 特征 → 模型 → 应用**

关键方法：

- 分子描述符（物理化学性质）
- 分子指纹（结构特征）
- 分子图表示（GNN）

模型体系：

- XGBoost / 随机森林
- 图神经网络（GNN）
- 深度学习模型

应用场景：

- 分子性质预测
- 虚拟筛选
- 反向设计

------

### 4. 大模型融合（RAG架构）

项目探索大模型与数据库的融合：

技术核心：

- Retrieval-Augmented Generation（RAG）
- 向量检索 + 结构化数据调用
- 工具调用（Tool Calling）

实现能力：

- 自然语言查询分子信息
- 分子结构解释
- 合成路径分析

目标：

> 构建“问答式化学数据库”
